### Утверждённый алгоритм отправки отчётов (22.06.2026)

**Установка (один раз):**
1. На ПК сотрудника копируется `WorkdayMonitor.exe`
2. При первом запуске сотрудник вводит свой логин
3. Настраивается Планировщик задач Windows (3 триггера)

**Ежедневная работа (автоматически, сотрудник ничего не делает):**

| Время | Что происходит |
|---|---|
| Утро, включение ПК | Приложение запускается. Скрипт проверяет — есть ли неотправленные отчёты за прошлые дни. Если есть — отправляет все на сервер |
| Весь день | Приложение работает в фоне, мониторит активность (программы, Битрикс, 1С, браузер) |
| 16:30 | Промежуточный срез — отчёт за сегодня отправляется на сервер |
| 23:00 | Полный отчёт за сегодня отправляется на сервер (перезаписывает срез 16:30) |

**Если ПК выключен до 23:00:**
- Срез 16:30 уже на сервере
- Полный отчёт отправится утром при следующем включении

**Если ПК не включался несколько дней:**
- При включении отправляются отчёты за все пропущенные рабочие дни
- Дни без данных (ПК не работал) — на дашборде прочерки "—"

**Что видит директор:**
- Заходит на http://46.149.68.148/login (пароль: порядок2026)
- Таблица: все сотрудники, начало/конец дня, активность, Битрикс, 1С, браузер
- Переключение дат стрелками, скачивание Excel-отчёта

**Перезапись, не дублирование:**
- Один логин + одна дата = одна строка
- Каждый новый отчёт за тот же день обновляет предыдущий
