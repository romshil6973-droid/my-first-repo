| `GET` | `/health` | Проверить доступность |

### Логика загрузки отчёта

```python
def upload_report(filepath: str, login: str) -> bool:
    """Загрузить файл отчёта на сервер."""
    url = f"{config.server_url}/upload/{login}"
    headers = {"X-API-Token": config.api_token}
    
    with open(filepath, "rb") as f:
        files = {"file": (os.path.basename(filepath), f, "application/octet-stream")}
        response = requests.post(url, files=files, headers=headers, timeout=30)
    
    return response.status_code == 200
```

### Когда происходит загрузка
- При завершении рабочего дня (кнопка "Завершить день")
- При выключении ПК (через системный хук)
- При ручном формировании отчёта

## 3. УСТАНОВЩИК PLUG & PLAY

### Концепция
Один файл `WorkdayMonitor_Setup.exe` — сотрудник запускает его, вводит только свой **логин** (никнейм), и приложение начинает работать автономно. Больше никаких действий не требуется.

### Технологический стек установщика
- **PyInstaller** — упаковывает Python-приложение в единый `.exe` (Python не нужен на ПК сотрудника)
- **Inno Setup** — создаёт профессиональный Windows-установщик

### Процесс установки (шаги пользователя)

```
1. Запустить WorkdayMonitor_Setup.exe
   ↓
2. Окно приветствия → [Далее]
   ↓
3. Ввести логин сотрудника:
   ┌─────────────────────────────┐
   │ Введите ваш логин:          │
   │ [shilov__________________]  │
   │                             │
   │ Только латинские буквы      │
   │ в нижнем регистре           │
   └─────────────────────────────┘
   [Установить]
   ↓
4. Автоматическая установка (прогресс-бар)
   ↓
5. Готово! Приложение запущено.
```

### Что делает установщик автоматически
- Копирует файлы в `C:\Program Files\WorkdayMonitor\`
- Создаёт `config.ini` с логином, URL сервера и токеном
- Прописывает автозапуск в реестре Windows
- Создаёт ярлык на рабочем столе
- Запускает приложение сразу после установки

### Структура файлов установщика
```
build/
├── WorkdayMonitor.exe         # PyInstaller bundle
├── installer.iss              # Inno Setup скрипт
└── WorkdayMonitor_Setup.exe   # Финальный установщик
```

## 5. ОБРАБОТКА ОШИБОК ПРИ ЗАГРУЗКЕ

### Сценарий 1: Сервер недоступен
```
1. Попытка загрузить отчёт → ошибка соединения
2. Сохранить файл локально: C:\Users\[user]\AppData\Local\WorkdayMonitor\pending\
3. Показать уведомление: "⚠️ Отчёт сохранён локально. Будет отправлен при восстановлении связи."
4. При следующем запуске — автоматически повторить загрузку
```

### Сценарий 2: Ошибка авторизации (401)
```
1. Залогировать ошибку
2. Показать: "❌ Ошибка доступа к серверу. Обратитесь к администратору."
3. Сохранить файл локально
```

### Сценарий 3: Повторная отправка накопленных файлов
```python
def retry_pending_uploads():
    """Отправить файлы из папки pending при восстановлении связи."""
    pending_dir = get_pending_dir()
    for filename in os.listdir(pending_dir):
        filepath = os.path.join(pending_dir, filename)
        login = config.get_login()
        if upload_report(filepath, login):
            os.remove(filepath)
            log.info(f"Повторно отправлен: {filename}")
```
