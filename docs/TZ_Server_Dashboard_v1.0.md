# ТЗ: Серверная часть и веб-дашборд «Система Порядка»
**Версия:** 1.0  
**Дата:** 17 июня 2026  
**Проект:** Система мониторинга рабочего времени  
**Сервер:** https://46.149.68.148 (Timeweb VPS, nginx, HTTPS)

---

## 1. Назначение

Серверная часть выполняет три задачи:
1. **Принимает** Excel-отчёты от WorkdayMonitor (уже реализовано)
2. **Парсит** отчёты и сохраняет метрики в базу данных
3. **Отображает** веб-дашборд для руководителей — таблица по всем сотрудникам за выбранный день

Дашборд открывается в браузере на любом устройстве (ПК, телефон, планшет).  
Отдельного десктоп-приложения нет — браузер и есть материнское приложение.

## 3. Файловое хранилище

### Структура папок

```
/opt/workday/
├── main.py              ← основной файл FastAPI
├── metrics.db           ← SQLite с метриками (хранится вечно)
├── config.py            ← пароль дашборда, настройки
└── uploads/
    ├── shilov/
    │   ├── 2026-06-17.xlsx   ← свежий, будет удалён через 30 р.д.
    │   ├── 2026-05-01.xlsx   ← [ARCHIVED] — не удалять
    │   └── ...
    ├── ivanov/
    │   └── 2026-06-17.xlsx
    └── .../
```

### Правила хранения Excel-файлов

| Тип файла | Срок хранения |
|-----------|--------------|
| Обычный отчёт | 30 рабочих дней, затем автоудаление |
| Архивный отчёт | Бессрочно (помечен вручную) |

**Маркер архивности:** файл переименовывается в `2026-05-01.archived.xlsx`  
(суффикс `.archived` защищает от автоудаления).

### Авто-очистка

При старте сервера и ежедневно в 02:00 запускается задача:
```
Для каждого файла в uploads/:
  Если имя НЕ содержит ".archived"
  И дата файла > 30 рабочих дней назад:
    Удалить файл
```

## 4. База данных метрик (SQLite)

### Таблица `daily_metrics`

```sql
CREATE TABLE daily_metrics (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    login       TEXT NOT NULL,
    date        TEXT NOT NULL,          -- 'YYYY-MM-DD'
    work_start  TEXT,                   -- '08:29:00'
    work_end    TEXT,                   -- '17:12:00'
    duration    INTEGER,                -- секунды
    active_time INTEGER,                -- секунды (без простоев)
    bitrix_time INTEGER,                -- секунды
    onec_time   INTEGER,                -- секунды (1С)
    browser_time INTEGER,               -- секунды
    achievements TEXT,                  -- текст из Листа 1 или NULL
    uploaded_at TEXT NOT NULL,          -- ISO timestamp загрузки
    UNIQUE(login, date)
);
```

**Метрики хранятся вечно.** Даже когда Excel-файл удалён через 30 дней,  
строка в `daily_metrics` остаётся — дашборд за любую историческую дату  
строится из базы мгновенно.

### Таблица `known_employees`

```sql
CREATE TABLE known_employees (
    login       TEXT PRIMARY KEY,
    first_seen  TEXT NOT NULL,   -- дата первого отчёта
    last_seen   TEXT NOT NULL    -- дата последнего отчёта
);
```

Список сотрудников формируется **автоматически**: при первом получении  
отчёта от нового логина он добавляется в таблицу. Ручной настройки нет.

## 5. Парсинг Excel-отчёта

Запускается **сразу после** успешного сохранения файла в `/upload/{login}`.

### Источник данных: Лист 2 «Мониторинг активности»

Структура строк:
```
(№, Начало, Конец, Продолжительность, Приложение / Операция, URL)
```

### Правила извлечения метрик

**Начало рабочего дня** — время первой записи в Листе 2 (строка с №=1).  
Дополнительно проверяем Лист 1, строка 3, ячейка B: `'Начало рабочего дня:'`.

**Конец рабочего дня** — время последней записи Листа 2 (поле «Конец»).

**Активное время ПК** — сумма длительностей всех строк,  
где поле «Операция» НЕ содержит `'Режим ожидания'` и `'Простой'`.

**Время Битрикс24** — сумма строк, где «Операция» содержит  
`'Bitrix'` или `'bitrix'` или `'итрикс'` (регистронезависимо).

**Время 1С** — сумма строк, где «Операция» содержит  
`'1cv8'` или `'1С'` или `'1с'`.

**Время браузеров** — сумма строк, где «Операция» начинается с  
`'browser'`, `'chrome'`, `'firefox'`, `'yandex'`, `'opera'`, `'edge'`, `'msedge'`.

**Достижения** — из Листа 1 «Фото рабочего дня»:  
строки 5 и далее, колонка «Операция», непустые значения, объединённые через `'; '`.  
Если Лист 1 пуст или не заполнен — значение `NULL` (в дашборде отображается `-`).

### Обработка опоздавшего отчёта

Если отчёт пришёл за прошедшую дату (например, в среду за пятницу):
- Метрики **обновляются** в БД (`INSERT OR REPLACE`)
- Исторический дашборд за ту дату автоматически отображает новые данные

```

```

## 7. Веб-дашборд

### Авторизация

- **Метод:** форма логин/пароль → cookie-сессия (24 часа)
- **Пароль:** один для всех руководителей, хранится в `config.py`
- Доступ только по HTTPS (nginx уже настроен)

### Структура страницы

```
┌─────────────────────────────────────────────────────────────────┐
│  📊 Система Порядка          [← 16 июн] [17 июня 2026] [→]     │
├──────────────┬───────┬───────┬──────┬──────────┬──────┬──────┬──────┬──────────────┬──────────┐
│ Сотрудник    │ Нач.  │ Кон.  │ Дл.  │ Актив.ПК │Битр. │  1С  │ Бр.  │ Достижения   │  Отчёт   │
├──────────────┼───────┼───────┼──────┼──────────┼──────┼──────┼──────┼──────────────┼──────────┤
│ shilov       │ 08:29 │ 17:12 │ 8:43 │  6:21    │ 2:14 │ 0:45 │ 1:10 │ Зарплата;... │[📥][📌] │
│ ivanov       │ —     │ —     │  —   │   —      │  —   │  —   │  —   │     —        │ Нет дан.│

└──────────────┴───────┴───────┴──────┴──────────┴──────┴──────┴──────┴──────────────┴──────────┘
│ [💾 Сохранить дашборд]   Обновлено: 11:47 · Сотрудников: 3 · Отчётов: 2 из 3      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

### Описание элементов

**Навигация по датам** (стрелки ← →):  
Переключает дашборд на предыдущий/следующий рабочий день.  
Даты без единого отчёта показываются серым и помечаются «Нет данных».

**Строка сотрудника:**
- Если отчёт есть — все ячейки заполнены, строка белая/светлая
- Если отчёта нет — все ячейки `-` или `—`, строка серая
- Время отображается в формате `Ч:ММ` (не `ЧЧ:ММ:СС`)

**Кнопка 📥 (Скачать):**  
Скачивает оригинальный Excel-файл.  
Если файл удалён (прошло >30 р.д.) и не заархивирован — кнопка неактивна, tooltip: «Файл удалён».

**Кнопка 📌 (Архивировать):**  
Помечает Excel-файл как постоянный (не удаляется автоматически).  
После нажатия иконка меняется на 📌✅, действие необратимо через UI.

**Кнопка «Сохранить дашборд»:**  
Скачивает текущую страницу как `.html` файл.  
Файл самодостаточен (стили встроены) — открывается без интернета.

**Строка статуса внизу:**  
`Обновлено: ЧЧ:ММ · Сотрудников: N · Отчётов: K из N`

### Адаптивность (мобильный телефон)

На экранах <768px:
- Таблица прокручивается горизонтально
- Колонки «Достижения» и «Отчёт» закреплены справа
- Шрифт уменьшается до 13px
- Навигация по датам остаётся вверху

## 8. Реестр сотрудников

Логика отображения дашборда:

```
1. Получить список ВСЕХ известных логинов из таблицы known_employees
2. Для каждого логина — запросить метрики за выбранную дату из daily_metrics
3. Если метрики есть → показать строку с данными
4. Если метрик нет → показать строку с прочерками (серая)
5. Сортировка: сначала те, у кого есть данные; затем — без данных (по алфавиту)
```

Новый сотрудник появляется в реестре **автоматически** при первой загрузке  
его отчёта. Никакой ручной регистрации не требуется.
